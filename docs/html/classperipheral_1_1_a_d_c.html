Detailed Description

template<class Method>
class peripheral::ADC< Method >

ADCクラス

ADCをDMA転送で読み続けるクラス
createChannel()とADC<Method>::Channelをうまく使うことをお勧めします

使用例
peripheral::ADC<F4::ADCMethod> adc(F4::adc1);
adc.config({ADC_Channel_0, ADC_Channel_1}, ADC_SampleTime_112Cycles, 1);
adc.read(ADC_Channel_0);
peripheral::ADC<F4::Method>::Channel ch;
if (auto tmp = adc.createChannel(ADC_Channel_0), tmp) {
ch = *tmp;
} else {
// 初期化エラー (adc.configで初期化していないチャンネルを指定した場合)
}
ch.readVoltage();

内部でのDMA転送について
少なくともSTMF3/F4のADC-DMAではADCの各チャンネルごとに初期化されたものを順に(順番は改変可能っぽい(未確認))メモリ転送するので、
ADC1-ch1, ADC1-ch2, ADC1-ch3を初期化した場合
ch1[t0], ch2[t0], ch2[t0], ch1[t1], ch2[t1], ch2[t1], ch1[t2], ch2[t2], ch2[t2], ...
という順番でメモリ転送されるみたい。なので、内部でVectorSliceとかいうクラスをこさえている(std::valarrayのスライスみたいな感じ)
Note
DMA転送の方式がマイコンによって異なる可能性があるので、そこをなんとかしなきゃね
