システム化の背景

RUR内のプログラムはPIDをそれぞれが作っていて無駄が多い.

そのうえ作られたPIDは制御工学的に問題が多い.

課題

PIDが人によって仕様が違う.

D制御を使っていない.

ワインドアップ現象の対策がされていない.

目的・方針

制御工学に基づいたPIDライブラリを作りRUR内のPID制御を統一する.

概要

マザボ,ミニコン,デカコン,PC等どのような環境でも使える.

ユーザーはパラメータを設定し制御量(現在値)と目標値を入力することでPID制御に従った操作量を得る.

機能

第1ステップ

D制御の部分にローパスフィルタを実装する.

ワインドアップ現象の対策を行う.

第2ステップ

測定値微分先行形PIDを構築する.

第3ステップ

さらなる高みへ

システム化する範囲

制御量は１つのPID制御.

一定周期で操作量をgetする関数が呼ばれることを前提として作る.

周期を一定にする部分はユーザーに保証してもらう.

ユーザーの入力ミス対策はここでは行わない.

移行計画

第1ステップ完成時点で足回り開発のPositionPIDに組み込む.

第2ステップ完成時点で強欲に組み込む.

作業標準

開発はgitflowで管理しているので従うように.

第2ステップまでのタグ付けは

完成しているステップ.開発中のステップ内でのリリース回数.全てのリリース回数

例えば第1ステップ開発中に機能が一つ完成してリリースするときは0.x.xとする.

第2ステップ開発中に機能が一つ完成してリリースするときは1.x.xとする.

0.0.0は何の工夫もされていないPID制御にする.
