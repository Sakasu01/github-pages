RUR_CANBoard_Master/interface

CAN基板送信側のインターフェースクラスです

各基板の入出力をCAN通信を通して操作できます

また，イベントを用いてマザボから特に指令を出すことなく制御することが可能です

Description

受信側(RUR_CANBoard_Slave)と関数を同じにするために基本的にRUR_CANBoard_Baseのインターフェースクラスを継承します．

基板クラスではインターフェースの操作に加え，基板内のイベントの設定関数も実装しています．

VS.

C++にしたので動作が分かりやすくなってる...かもしれない

ここのインターフェースは全基盤共通にできるので整備性が良くなった

Requirement

  • CANProtocol (C++のやつ)
  • RUR_CANBoard_Base
  • RUR_STM_PeriphLibなど，CANをいじるもの

Usage

サブモジュールとしてuserlibに追加してください

board::sender::ControllerBoard cb(0);
constexpr board::controller_board::InitParameter initParam = {
board::controller_dc::Mode::undifined, //誤動作防止のため未初期化，MDに合わせてください
std::array{
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::adc,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
board::adio::Mode::gpioIn,
}
};
cb.config(initParam);

{
it.setState(true);

}

it.setState(true);
it.setPulse(1000.f);
}
