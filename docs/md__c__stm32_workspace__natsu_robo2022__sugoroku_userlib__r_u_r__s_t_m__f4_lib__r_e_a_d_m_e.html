割り込みについて

ライブラリ内で使用される割り込み関数はすべてライブラリ内部で使用されているため、ライブラリの外で宣言しなくてもよいです

但し、処理速度の向上などを理由にライブラリの外部で割り込みハンドラを宣言、定義することは可能です(ライブラリ内の割り込みハンドラは__attribute__((weak))が指定されている)

その場合はライブラリ側の割り込みが機能しなくなるので、その点に注意してください

割り込みハンドラの定義の方法

割り込みハンドラを宣言、定義するファイルがC++としてコンパイルされる場合(デフォルトでは拡張子が.cppのファイル)はコンパイラが関数名に型情報を埋め込む(mangling)されることを防がなければなりません

そのためにはリンゲージ指定子を用います

// リンゲージ指定子 この範囲の宣言、定義はすべてC言語とおなじ名前規則が適用される
extern "C" {
void XXX_IRQHandler(void) {
...
}
}
